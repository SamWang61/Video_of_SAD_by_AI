# AI Animation Universe Engine v2.0

v2.0 將單一 Gemini / Veo Prompt 擴充為跨平台內容生產流程。

## 生產管線

```text
角色定稿
  → 場景參數化
  → 影片生成
  → 循環品質檢查
  → 音效與配音
  → 社群互動設計
  → 發布與成效回饋
```

## 模組

### 1. 角色定稿

適用：ChatGPT Images、Leonardo AI 或其他影像工具。

輸出：

- 正面主視覺
- 表情基準
- 服裝與色票
- 身材比例
- 角色一致性參考圖

### 2. 循環影片生成

適用：Gemini / Veo、Kling、Runway、Luma。

固定規格：

- 9:16
- 1080 × 1920
- 4 秒
- 24 fps
- 固定鏡頭
- 1–3 秒動作週期
- 首幀與尾幀一致

### 3. 聲音設計

可使用模型原生音訊，或後製加入：

- 環境底噪
- 與重複動作同步的節奏音
- 遠近層次音效
- 必要時的配樂或旁白

音訊的節奏應在 2 倍速播放時產生更明顯的喜劇反差。

### 4. 社群互動設計

使用平台原生長按倍速功能，而非把紅點誤認為真正的程式按鈕。

建議文案：

- 長按紅點，幫他加速。
- 按住不要放，他快撐不住了。
- 長按 2 倍速，讓 KPI 追不上他。

### 5. 平台輸出

| 平台 | 建議輸出 |
|---|---|
| Instagram Reels | 9:16 MP4、短文案、紅點位於安全區 |
| Facebook Reels | 9:16 MP4、清楚操作提示 |
| TikTok | 9:16 MP4、首秒即顯示長按提示 |
| YouTube Shorts | 9:16 MP4、簡短標題與循環結構 |

## 品質閘門

影片未通過以下條件，不進入發布：

1. 角色一致性通過。
2. 動作閉環通過。
3. 首尾銜接通過。
4. 音訊無爆音與突切。
5. 正常速度與 2 倍速都能理解。
6. 紅點沒有被平台 UI 遮住。
7. 文案明確說明「長按」。
