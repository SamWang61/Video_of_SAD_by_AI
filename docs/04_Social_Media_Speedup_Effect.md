# 社群媒體長按加速效果
# Social Media Long-Press Speed-Up Effect

## 核心原理｜Core Principle

畫面中的紅點是視覺引導，不是影片內真正的互動按鈕。觀眾長按短影音畫面時，支援此功能的平台會啟動原生倍速播放，使影片畫面與內嵌音訊同步加速。

The red dot is a visual cue, not a real interactive button embedded in the video. When viewers long-press a short-form video on a platform that supports this behavior, native accelerated playback speeds up both the video and its embedded audio.

```text
紅點與文案引導 / Red dot and instruction
    → 觀眾長按畫面 / Viewer presses and holds
    → 平台啟動倍速 / Platform activates accelerated playback
    → 動作與音效同步加快 / Motion and sound speed up together
    → 形成反差式幽默 / Contrast-based comedy appears
```

## 製作原則｜Production Principles

1. 正常速度應稍慢，讓倍速反差明顯。  
   Normal playback should be slightly slow so the speed-up contrast is obvious.
2. 動作要簡單、週期性強，避免倍速後難以辨識。  
   Use simple, strongly periodic actions that remain readable at higher speed.
3. 音效應與動作同步，例如槳聲、鍵盤聲、腳步聲。  
   Synchronize sound effects with movement, such as oar strokes, typing, or footsteps.
4. 紅點應放在上方安全區，避開平台的愛心、留言與分享介面。  
   Place the red dot inside the upper safe zone and away from platform interface elements.
5. 文案必須明確寫「長按」，不要只寫「點一下」。  
   Tell viewers to “press and hold,” not merely to “tap.”
6. 正常速度與加速後都要有笑點。  
   The scene should work comedically at both normal and accelerated speed.

## 推薦引導文案｜Suggested On-Screen Instructions

- 長按紅點，幫他加速。  
  Press and hold the red dot to speed him up.
- 按住不要放，工作還沒做完。  
  Keep holding. The work is not finished yet.
- 長按加速，讓他逃離星期一。  
  Hold to help him escape Monday.
- 幫他快一點，不然下班會變成都市傳說。  
  Speed him up, or leaving work will become an urban legend.

## 幽默設計矩陣｜Comedy Design Matrix

| 正常速度 Normal speed | 加速後 Accelerated playback | 笑點 Comedy payoff |
|---|---|---|
| 慢慢划船 Slow rowing | 像在緊急逃生 Emergency rowing | 明明仍在原地，努力感卻翻倍 Still going nowhere, but working twice as hard |
| 無奈打字 Tired typing | 月底報表大爆炸 Month-end report chaos | 職場焦慮被具象化 Workplace anxiety becomes visible |
| 超慢跑 Very slow jogging | 像被 KPI 追殺 Chased by KPIs | 速度與厭世表情完全不匹配 Speed and expression hilariously conflict |

## 發布前實測｜Pre-Publication Testing

- 使用正式發布後的 Reel、Short 或 TikTok 頁面測試。  
  Test the actual published Reel, Short, or TikTok post.
- 分別使用 iPhone 與 Android。  
  Test on both iPhone and Android.
- 使用自己的帳號與另一個帳號。  
  Test with the publishing account and another viewer account.
- 確認長按不會只觸發暫停。  
  Confirm that long-pressing does not merely pause playback.
- 確認平台 UI 隱藏後，紅點與角色仍可辨識。  
  Confirm that the red dot and character remain visible when the interface changes.
- 確認音訊會跟著倍速，而不是被平台靜音或替換。  
  Confirm that embedded audio speeds up rather than being muted or replaced.

## 風險與替代方案｜Risks and Alternatives

平台功能可能因版本、地區、裝置或帳號測試而不同。若平台不支援長按倍速，可輸出一支預先剪輯的「正常速度 → 加速」版本。

Platform behavior may vary by app version, region, device, or account experiment. When native long-press acceleration is unavailable, create a pre-edited version that automatically changes from normal speed to accelerated playback.

- 原生互動版：由觀眾長按觸發。  
  Native interactive version: the viewer triggers acceleration by pressing and holding.
- 預剪加速版：影片在固定時間自動加速。  
  Pre-edited version: the clip accelerates automatically at a predetermined moment.
